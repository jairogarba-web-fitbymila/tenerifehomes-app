#01 Crítico Wizard de reservas: "No hay huecos disponibles" en todas las fechas Booking ▾
Problema

Al intentar reservar online, el wizard muestra "No hay huecos disponibles" para TODAS las fechas, incluso las que tienen citas reales en el panel admin. Bloquea el 100% de las reservas online.

Impacto
🔴 Bloquea ventas — 0% conversión online
Causa raíz probable

El wizard de reservas usa una lógica de disponibilidad diferente (code path distinto) al que usa el asistente de voz Vapi. Vapi sí puede reservar correctamente, pero el wizard web no encuentra slots.

Solución propuesta

Unificar la lógica de disponibilidad del wizard con la de Vapi Tools. Verificar la query que busca available_slots para la fecha seleccionada — posible problema con timezone o formato de fecha. Comparar queries de /api/booking/availability vs /api/vapi/check-slots.

#02 Alta Roles del equipo en inglés: "owner", "stylist" en vez de español Admin → Equipo ▾
Problema

Los roles del equipo se muestran en inglés ("owner", "stylist") tanto en el panel admin como en la web pública. Para un SaaS enfocado en España, esto confunde a los dueños de salones que no hablan inglés.

Impacto
🟠 UX confusa — aspecto poco profesional
Solución propuesta

Crear un mapa de traducción de roles: owner → Propietaria, stylist → Estilista, receptionist → Recepcionista, etc. Aplicar en la tabla team_members o en la capa de presentación.

#03 Alta Formato de moneda incorrecto: "15EUR" en vez de "15 €" Web pública ▾
Problema

Los precios se muestran como "15EUR" sin espacio y con código ISO. El formato español correcto es "15 €" (cifra + espacio + símbolo). Afecta a toda la sección de servicios y al wizard de reserva.

Impacto
🟠 Aspecto amateur — poca confianza del cliente
Solución propuesta

Usar Intl.NumberFormat('es-ES', { style: 'currency', currency: 'EUR' }) o un helper formatPrice(amount) que devuelva siempre "XX €". Aplicar globalmente.

#05 Alta Calendario permite seleccionar días cerrados Booking ▾
Problema

En el wizard de reserva, el calendario permite seleccionar cualquier día incluyendo los que el salón tiene marcados como cerrados (domingos, festivos). Debería deshabilitar y griselar esos días.

Impacto
🟠 Frustración del cliente — intenta reservar y no hay horas
Solución propuesta

Consultar salon_schedules para obtener los días operativos del salón. Pasar como disabledDays al componente de calendario. Los días cerrados se muestran en gris y no son clicables.

#06 Alta Clientes duplicados por formato de teléfono Admin → Clientes ▾
Problema

El sistema crea clientes duplicados cuando el mismo teléfono se introduce con formatos distintos: "612345678", "+34612345678", "612 345 678". No hay normalización al guardar.

Impacto
🟠 Datos sucios — historial fragmentado del cliente
Solución propuesta

Normalizar teléfonos al guardar: eliminar espacios, guiones, paréntesis. Si no tiene prefijo internacional, añadir +34 por defecto. Usar unique constraint en phone_normalized. Merge de duplicados existentes.

#07 Media Avatares del equipo solo muestran iniciales, sin foto Admin → Equipo ▾
Problema

Los miembros del equipo muestran solo un círculo con iniciales en lugar de su foto de perfil. No hay opción de subir foto en el formulario de edición del equipo.

Impacto
🔵 Aspecto genérico — pierde personalización
Solución propuesta

Añadir campo avatar_url a team_members y un input de upload de imagen en el formulario de equipo. Subir a Supabase Storage en el bucket del salón. Fallback a iniciales si no hay foto.

#08 Media CTA solo ofrece "Llamar", falta "Reservar online" Web pública ▾
Problema

El botón principal de acción (CTA) en la web pública solo ofrece "Llamar". Debería incluir un botón prominente de "Reservar Online" que lleve al wizard de reserva — ese es el objetivo principal del SaaS.

Impacto
🟠 Pierde reservas online — principal fuente de valor
Solución propuesta

Añadir botón CTA principal "Reservar Cita" que enlace a /{slug}/reservar. El botón de "Llamar" pasa a secundario. Considerar también un botón flotante sticky para móvil.

#09 Media Sección de ubicación sin mapa de Google Maps Web pública ▾
Problema

La sección de ubicación/contacto en la web pública muestra la dirección como texto pero no incluye un mapa interactivo. Los clientes necesitan saber cómo llegar.

Impacto
🔵 Pierde contexto visual — el cliente no sabe dónde está
Solución propuesta

Embed de Google Maps iframe usando la dirección del salón. Almacenar latitude y longitude en salons o usar embed de Google Maps con dirección. No requiere API key si se usa iframe embed.

#10 Media Confirmación de reserva promete email que no se envía Booking ▾
Problema

La pantalla final del wizard dice "Recibirás una confirmación por email" pero el email no llega (o no se envía). Genera desconfianza en el cliente que no sabe si su reserva se registró.

Impacto
🟠 Desconfianza — cliente no sabe si se reservó bien
Solución propuesta

Verificar el flujo de email con Resend. Si el email funciona, comprobar que se captura el email del cliente en el wizard. Si no está implementado, o quitar el mensaje o implementar el envío con template HTML profesional.
